# LLM Tokenizer

In this lab we process raw text into a form that a language model can learn from. The data begins as plain text (e.g., Frankenstein chapters), and we first read it into a Python variable `raw_text`. Tokenization is the step that turns this stream of characters into discrete units, called tokens, which the model can convert to numbers. This is necessary because neural networks are numerical, and text must be mapped into a numerical format in order to be fed through embeddings and trained.

Tokenization is a necessary step for LLMs because human language includes punctuation, whitespace, and different words in many forms. The code in `LLMLab1Starter.ipynb` uses a regular expression to split on punctuation (`([,.:;?_!"()\']|--|\s)`), double dashes, and whitespace. It then filters out empty chunks and control characters like `\n` and `\t`. This creates a sequence of tokens that preserve the meaning of the original text while standardizing word boundaries, which is required to build a deterministic vocabulary.

The lab then removes duplicate tokens using `sorted(set(preprocessed))`, which creates the vocabulary of unique tokens. Each unique token is assigned an integer ID. These token IDs are the core language model input representation; they allow the same text to be converted to a fixed numeric space where each token has a stable index. `SimpleTokenizerV1` wraps this process in a Python class. The `encode` method applies the same spacing-and-punctuation regex splitting to new text, removes empty tokens, and replaces each token with its integer ID. The `decode` method reverses the mapping and joins tokens back into a relaxed text string.
