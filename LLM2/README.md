# LLM Architecture Explanation

The LLM architecture is used in LLMLab2.ipynb builds on the attention mechanisms we talked about in class. Instead of using just one hidden state at a time like RNNs do, it gives different parts of the input different weights to make context-aware representations. The lab starts by making a configuration dictionary called GPT_CONFIG_124M. This dictionary sets important settings for the query-key-value parts, such as the size of the vocabulary, the length of the context, the number of attention heads, the number of layers, the dropout rate, and the bias settings. This setup is what you need to build the GPTModel class, which includes the main parts of a transformer-based architecture, such as the attention mechanisms that let the model compare the importance of different input tokens.

The GPTModel uses token and positional embeddings to turn input tokens into vectors and add positional information to keep the order of the sequence. This is what we talked about when we talked about how embeddings make context-aware processing possible. The model then uses a series of transformer blocks, each of which has attention layers that use dot products between input vectors to figure out attention scores, softmax to make probability distributions, and context vectors that show how each token relates to the words around it. This process directly uses the simplified self-attention and causal attention ideas from the lecture. It does this by giving weights to input vectors to make a context vector that captures the subtleties of language beyond simple one-to-one mappings. This makes sure the model can handle complex sequences.

The architecture ends with a forward pass that sends input batches through the embeddings, transformer blocks, layer normalization, and an output head to make logits for the next token predictions. The lab demonstrates this by employing random weights to construct the model and executing it on encoded text batches. The generate_text_simple function demonstrates the model's practical application by utilizing context vectors to make predictions and incrementally add new tokens. This random weights is why when we try to generate text the end result is just a bunch of nothing.
